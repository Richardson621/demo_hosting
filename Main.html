Design Documentation

User-Centered Design Evidence

Persona: Alex, a 20-year-old undergraduate student majoring in Computer Science. Alex is tech-savvy but often busy with multiple classes and extracurriculars, needing a quick and intuitive way to submit assignments and check statuses on the go via mobile.

Scenario: Alex receives a notification about an upcoming assignment deadline for DCIT 202. They open the mobile app, log in quickly, select the course, upload the file, and immediately check the status to confirm submission before heading to class.

User Story: As a student, I want to submit assignments and view their statuses on the same screen so that I can manage my coursework efficiently without navigating multiple pages.

Application of HCI Principles

  • Consistency: Uniform button styles, navigation, and color schemes across all screens and between mobile app and website ensure users feel familiar navigating the interfaces.
  • Visibility: Key actions like 'Submit' buttons and status indicators are prominently placed with clear labels and icons, with the merged submission and status screen reducing navigation steps.
  • Feedback: Status tags (e.g., Submitted, Pending) use color-coded badges for immediate visual feedback, and forms include placeholders and required fields to guide users during input.
  • Accessibility: ARIA labels, keyboard navigation (tabindex), focus outlines, and semantic HTML improve usability for screen readers and keyboard users.

Reflection

If I had more time, I would conduct usability testing with students, measuring task completion times for submitting and checking assignment statuses on the merged screen, error rates, and satisfaction via surveys. I'd also perform A/B testing on the merged layout versus separate screens and accessibility audits with tools like WAVE.
